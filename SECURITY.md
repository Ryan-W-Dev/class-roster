# Security policy

## Supported version

Security fixes are applied to the latest release on the `main` branch.

## Reporting a vulnerability

Please report suspected vulnerabilities through GitHub's private security-advisory feature. If private reporting is unavailable, contact the maintainer privately through the [LinkedIn profile](https://www.linkedin.com/in/ryan-wynn-01a784403/) before sharing technical details publicly.

Do not include real names, military IDs, roster files, or other sensitive information in an issue, screenshot, or sample file.

## Data handling

Class Roster operates locally and does not transmit records to a server. It writes an unencrypted Excel workbook to `Documents/Personnel Management System/database.xlsx`. Users remain responsible for device security, file permissions, backups, retention rules, and any organizational policies that apply to the stored information.
